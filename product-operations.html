Product and operations

System architecture, feature set and delivery cost

This page reworks the source document's product-definition and costing chapters into a web-native format.

Product overview

The proposed product is a real-time remote monitoring and emergency control system for automated vehicles. It is intended to improve safety, reliability and operational efficiency in controlled environments by combining autonomous trajectory tracking with remote supervision.

The product is framed as an intermediate architecture. Vehicles operate autonomously under normal conditions, while a remote operator can monitor state, review perception inputs and intervene in abnormal scenarios.

Product positioning

Lightweight, cost-effective and designed for campuses, business parks, low-speed routes and other controlled environments where full continuous teleoperation is unnecessary but safety oversight remains important.

System architecture

Four major subsystems

Vehicle subsystem

Includes cameras, LiDAR, localisation hardware and onboard data generation for position, speed and scene perception.

Control system

Implements longitudinal and lateral control to maintain accurate trajectory tracking and stable behaviour.

Remote monitoring interface

Human-machine interface for vehicle state, camera feeds, trajectory information, alerts and intervention controls.

Communication layer

Bidirectional low-latency data flow between the vehicle and the operator, with validation and consistency checks.

Operating concept

How the system is intended to work

  1. Sensor data is collected and processed onboard the vehicle.
  2. The control system maintains trajectory tracking.
  3. Vehicle and sensor state are transmitted to the remote operator interface.
  4. The operator supervises performance in real time.
  5. If an anomaly occurs, the operator can intervene through emergency braking or override commands.

Normal mode

Autonomous trajectory following on predefined routes.

Supervision mode

Human-in-the-loop oversight through a remote HMI and live data feeds.

Intervention mode

Selective remote action in safety-critical or abnormal conditions.

Functional requirements

Core, safety and communication features

Core functional features

  • Trajectory tracking with longitudinal and lateral control
  • Real-time monitoring of vehicle and sensor data
  • Remote supervision through a dedicated HMI

Safety features

  • Emergency braking available to the operator
  • Fail-safe mechanisms for communication or control failure
  • Redundancy checks to improve integrity and reliability

Communication features

  • Low-latency streaming with a target below 200 ms
  • Bidirectional communication for control and feedback
  • Data integrity checks to reduce information loss

Cost model

Estimated product development cost

Personnel cost

The document estimates total personnel cost for seven months at approximately £237,000.

Software infrastructure

Cloud compute, storage, monitoring and development tools are grouped at approximately £5,000.

Testing and validation

Field testing, simulation, data analysis, debugging and deployment preparation total approximately £17,000.

Hardware and prototype cost

Rounded to £36,000 in the final total, based on line items covering cameras, LiDAR, GNSS/INS, onboard compute, drive-by-wire and networking.
